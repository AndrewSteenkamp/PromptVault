### Lesson 2.1: The Automation Architecture

Every automation follows this pattern:

**TRIGGER** (What starts it) → **CONDITIONS** (When to run) → **ACTIONS** (What to do) → **RESULTS** (What happens)

**Example 1: Lead Capture Automation**

Trigger: "New form submission on website"
Condition: "If email is not blank"
Action 1: "Add to email list"
Action 2: "Send welcome email"
Action 3: "Create CRM entry"
Result: Lead is captured and welcomed

**Example 2: Invoice Automation**

Trigger: "New order in Shopify"
Condition: "If order total > $100"
Action 1: "Generate invoice"
Action 2: "Send invoice email"
Action 3: "Update accounting software"
Action 4: "Create shipping label"
Result: Order is processed automatically

**Example 3: Content Distribution Automation**

Trigger: "New blog post published"
Condition: "If category = 'Marketing'"
Action 1: "Create social media post"
Action 2: "Send email to subscribers"
Action 3: "Post to LinkedIn"
Action 4: "Post to Twitter"
Result: Content is distributed across all channels

### Lesson 2.2: Triggers, Conditions, and Actions

**TRIGGERS (What starts the automation)**

Common triggers:
- New form submission
- New email received
- New database entry
- Scheduled time (daily, weekly, monthly)
- Webhook event
- File uploaded
- API call received

**CONDITIONS (When to run)**

Conditions filter which automations run:
- IF email contains "@company.com" → run automation
- IF amount > $100 → run automation
- IF status = "approved" → run automation
- IF date is between X and Y → run automation

**ACTIONS (What happens)**

Common actions:
- Send email
- Create database entry
- Update spreadsheet
- Post to social media
- Generate document
- Call API
- Send Slack message
- Create calendar event

### Lesson 5.1: The Advanced Prompt Formula

**Basic Formula:**
[ROLE] + [CONTEXT] + [TASK] + [CONSTRAINTS] + [OUTPUT FORMAT]

[ROLE] + [CONTEXT] + [TASK] + [CONSTRAINTS] + [EXAMPLES] + [TONE] + [OUTPUT FORMAT]

```

CONTEXT: I sell project management software to small business owners (5-50 employees)

- Each email 150-200 words
- Focus on pain points: wasted time, disorganized teams, missed deadlines
- Include social proof in email 3
- CTA in every email
- Personalization variables: [NAME], [COMPANY], [PAIN_POINT]
EXAMPLES:
Email 1 subject: "How [COMPANY] is wasting 10 hours per week"
Email 2 subject: "3 founders who fixed their team chaos"
TONE: Friendly, conversational, not salesy
OUTPUT FORMAT: 
Email 1: Subject line + Body
Email 2: Subject line + Body
...

Target audience: Busy entrepreneurs, 5-50 employees
Email 1 goal: Introduce solution
Email 2 goal: Show social proof
Email 3 goal: Overcome objections
Email 4 goal: Create urgency
Email 5 goal: Final CTA
```

### Lesson 5.2: System Prompts

System prompts set the behavior for the entire conversation.

**Example System Prompt:**

```
You are an expert business consultant with 20 years of experience helping startups scale from $0 to $1M+ in revenue.

Your communication style:
- Direct and actionable
- Focus on practical, implementable advice
- Use data and examples
- Challenge assumptions respectfully
- Ask clarifying questions when needed

When providing advice:
1. Understand the current situation
2. Identify the core problem
3. Provide 3 solutions with pros/cons
4. Recommend the best solution with implementation steps
5. Provide metrics to track success

Always prioritize:
- Revenue impact
- Time to implement
- Cost-benefit analysis
- Scalability
```
